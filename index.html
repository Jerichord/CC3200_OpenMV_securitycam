CC3200 OpenMV Security Camera

A simple security system that sends a base-64 message to AWS, which the user can then decode to see who entered their home.

Microcontroller

With the CC3200, track the movement of your door using the built-in accelerometer,alert a camera to turn on using UART, and send a POST request to Amazon containing an encoded image.

Camera

Using the OpenMV camera, capture who opened your door using a facial tracking program, and send a base-64 encoded image back to the CC3200 using UART.

AWS

By using digital certificates, connect devices as "things" and set up a simple notification service that will update a user whenever a topic is triggered.

Email/SMS

Connect your email and be notified with an encoded message that you can run through a decoder and see a picture of who is at your door.

Approach

To communicate between the CC3200 and the OpenMV camera we only need a:

Signal
Tells the OpenMV when to turn on the camera and start the image detection algorithm to take a picture
Protocol
To be able to send the base64 encoded from the OpenMV to the CC3200.

The other protocols such as SPI and I2C were too complicated to implement in the amount of time we had and required porting over a python based library to the CC3200. UART met all of our needs above and is a fairly fast and easy protocol to take advantage of, requiring only 3 wires. We used it to send a string to the OpenMV to tell it to start the camera and for the OpenMV to send the picture, once it had taken it, back to the CC3200 in a quick manner.

Uses

The accelerometer on the CC3200 has a sensor which can read x, y, and z values. We will mount the board on the door, so the values that we read will be the z-axis, represented by the red arrow. By watching a threshold on this value, we can easily set off a chain of events that will happen, namely starting up the camera, sending an http post and start a facial detection program, which will send its result back

Improvements/Practicality

Our system is very immature, but some obvious improvements include a more ideal mount for the board, either through installment inside a frame, or using a different measurement in general so that there will not be a long wire needed. On the virtual end, we can write a script that decodes the base-64 into an image automatically, and sends the image directly to AWS, and may store it inside its S3 storage service.
